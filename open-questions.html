The eight questions, in detail

Working slides
Question 3 Category · Resources Priority · Urgent Owner · Lead

Activate the partner-side resources.

This is the most time-sensitive of all eight open questions. Without this meeting, the donation pipeline stalls — and that feature sits on the critical path for launch.

  • Azure nonprofit credits — $2,000 per year, auto-renewed, currently not realized. These credits fund the hosting infrastructure for the app. Activation requires a coordination meeting with the appropriate partner-side contact.
  • Zeffy API key — required for the donation integration. Without this key, attendees cannot donate through the app. Activated in the same partner-side meeting.
Question 1 Category · Scope Priority · High Owner · Board

Geographic scope of map content.

How far should the app's map reach? Each answer has a different scope impact on the six-week build.

  • Festival grounds only?
  • Colorado Springs + Manitou Springs + Fountain only?
  • Full directory — include Denver + Pueblo (all 104 places)?
Question 2 Category · Scope Priority · High Owner · Board

Place detail depth per listing.

How deep should each directory listing go?

  • Public website info only per location?
  • Or custom content per location — story, hours, accessibility notes?

Note: The festival doesn't have social-media accounts, so any custom content would need to be generated directly by Juneteenth staff or business owners. That's a content-authoring lift, not just a sourcing question.

Question 5 Category · Scope Priority · Medium Owner · Board

Additional app content beyond the basics.

We have the festival basics: map, schedule, vendors, food trucks, sponsors. What else should be available in the app? Examples to react to (these are prompts, not proposals): festival history, daily highlights, accessibility info, transportation, a kids' track. The board's "what else?" sets the rest of the content scope.

Question 6 Category · Engagement Priority · High Owner · Board

Include the ALAI "Did You Know" experience?

ALAI — Digital Ally Project's AI tutoring agent — is an intelligent tutoring system that captures knowledge levels per person across content libraries. Should the app include a "Did You Know" experience powered by ALAI?

This is arguably the most important interaction the cohort can capture. The data signal of what learners absorb at the festival is core to Digital Ally Project's mission. See Meet ALAI for the full picture of what's possible.

Question 7 Category · Engagement Priority · High (if Q6 = yes) Owner · Board

If yes to Q6 — who creates the content?

ALAI needs curated content libraries: Juneteenth history, Black-heritage sites in Southern Colorado, performer backgrounds, local figures honored at monuments. We need a designated content owner who can make this happen on the six-week timeline.

Candidates the board may want to consider: a staff member, a community historian, a partnership with one of the directory's museums or cultural institutions, a volunteer team. The cohort can scaffold the structure; the content itself needs a human owner.

Question 9 Category · Engagement Priority · Medium Owner · Board

Voice or text chat for the festival assistant?

Is a text-based chat interface sufficient for the festival context, or do attendees need voice interaction with the festival assistant? The answer affects both build scope and accessibility.

  • Text only — simpler, faster to ship, works on every phone.
  • Text + voice — closer to "talk to a knowledgeable friend," more accessible for visually-impaired attendees, more work to build.
Question 8 Category · Privacy Priority · Medium Owner · Board

Data minimization floor.

What festival-goer data absolutely needs to be collected? The internet has moved toward privacy-first design, and we want to default to collecting less, not more.

Board input here sets the floor. The less we collect, the less we're responsible for protecting — and the more trust we build with attendees. The cohort recommends starting with: account email (only if they sign in), and the engagement signals ALAI needs to personalize (if Q6 = yes), with everything else opt-in.

Install on your device

Add this to your home screen

This site can run like an app on your phone — full-screen, offline-capable, no app store required.

iPhone or iPad (Safari)
  1. Tap the Share button (square with an up-arrow) at the bottom of Safari.
  2. Scroll down and tap Add to Home Screen.
  3. Tap Add in the top-right.
Android — use Chrome (not Edge)
  1. Open this page in Chrome on Android.
  2. Tap the three-dot menu in the top-right.
  3. Tap Install app (or Add to Home Screen).
  4. Tap Install to confirm.

Important: on Android, install with Chrome — not Edge. Chrome creates a standalone app that uninstalls cleanly. Edge installs a shortcut bound to the Edge browser, so removing the icon can prompt to uninstall Edge itself.

Desktop (Chrome or Edge)
  1. Look for the install icon in the address bar (a small monitor with a down-arrow).
  2. Click it, then click Install.

Heads up before you uninstallTo remove this app later: on Windows desktop, open edge://apps/ (Edge) or chrome://apps/ (Chrome) and uninstall from there — not from Windows Apps & Features, which can accidentally remove the browser. On Android, long-press the icon and tap Uninstall — but only if you installed via Chrome. Edge on Android stores PWAs as browser-bound shortcuts, so uninstalling can remove Edge itself. On iPhone / iPad, long-press the icon, tap Remove App, then Delete from Home Screen — safe by design.
